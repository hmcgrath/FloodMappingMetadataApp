Flood Mapping Metadata Portal

About

The flood mapping metadata portal is an online database to (i) enter and (ii) query metadata related to flood hazard mapping. This site was created by Natural Resources Canada in collaboration with Conservation Ontario.

Data Entry:

This form is intended to collect metadata that describes the data, inputs, and models used to develop flood maps.

The default data that is required includes details about the type of record (hazard, awareness, etc.), the project category (e.g.: watercourse, great lakes, coastal, etc.), name of water body, hazard events evaluated (e.g.: 100 yr, 20 yr, 5 yr, etc.). The spatial extent of the area mapped can be loaded multiple ways: via mouse-clicks or uploading a file of the flood line polygon or extent of mapped area.

On the 2nd page, default required information includes: financial support, new/updated mapping, the area covered (e.g.: length of waterbody, maximum floodplain extent, etc.) and if a flood event has been recorded since 1996. In addition to the default data required, users may select any (or all) of the buttons below to add additional information they have available regarding the data used in the modelling process:

  • Imagery Data - For recording information on images used in the development of the flood map.
    • Data entry fields include: season of collection, acquisition year, horizontal and vertical reference datum, accuracy and spatial resolution.
  • Elevation Data - For recording information on elevation dataset(s) used in the development of the flood map.
    • Data entry fields include: data type (e.g.: LiDAR, GPS, phogrammetry, etc.), data format, acquisition year, horizontal and vertical reference datum, accuracy and spatial resolution.
  • Hydrology Data - Hydrology Data : For recording information on hydrology models and methods used.
    • Data entry fields include: hydrology method used, number of years in datasets, number of events modelled, quality of the input and calibration, if future model year events are modelled, climate change considerations, and whether snowmelt, peak volume reduction or catchments were discretized.
  • Hydraulics Data - For recording information on hydraulic model and calibration process used.
    • Data entry fields include: hydraulic modelling software used, model year used, flow conditions, quality of the input and calibration, whether the floodline was estimated, if the results have been peer reviewed and any general comments.
